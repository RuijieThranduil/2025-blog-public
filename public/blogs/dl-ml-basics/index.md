# DeepLearning · ML Basics 机器学习底座

深度学习是机器学习的一部分，不是和机器学习平行的另一门学科。  
所以很多 deep learning 里的问题，其实先要用 ML 的语言去看。

## 1. 监督学习到底在做什么
最朴素的说法是：
- 给输入
- 给标签
- 学一个从输入到输出的映射

但更准确一点，它是在有限样本上拟合一个规律，并希望这个规律在没见过的数据上也成立。

也就是说，关键从来不只是 fit，而是 generalize。

## 2. Loss function 连接了目标和训练
模型输出本身没有意义，只有当你定义了 loss，它才知道什么叫“错”。

常见例子：
- 分类常用 cross entropy
- 回归常用 MSE / MAE

所以 loss 不只是评价函数，它其实在定义训练目标的几何形状。

## 3. 偏差和方差是理解模型表现的老钥匙
一个很好用的框架是：
- bias 高：模型太简单，学不住
- variance 高：模型太敏感，泛化差

这套语言今天看起来很“传统”，但它仍然能解释很多深度学习现象：
- 为什么会过拟合
- 为什么更大模型有时反而更稳
- 为什么 regularization 有意义

## 4. 评估模型时，别只盯 accuracy
不同任务里，单一指标可能非常误导。

比如分类问题里，还可能需要看：
- precision
- recall
- F1
- ROC / AUC

如果数据不平衡，只看 accuracy 往往会高估模型。

## 5. 训练集表现好，不代表模型真的强
这是很多初学者最容易踩的坑。  
训练误差低，只能说明模型把训练集学得不错。

真正要看的通常是：
- validation / test 上的表现
- 泛化误差
- 对数据分布变化的鲁棒性

这也是为什么数据划分、交叉验证、模型选择都重要。

## 6. 为什么 ML Basics 对 DeepLearning 依然关键
因为 deep learning 只是用更强表达能力的模型，去回答同样的老问题：
- 什么是好模型
- 怎么定义误差
- 如何防止过拟合
- 如何评估泛化

如果这些基本问题没想清楚，模型再大也只是更复杂地犯错。

## One-line takeaway
很多“深度学习问题”往下挖一层，其实还是机器学习最经典的建模、泛化和评估问题。
