# LLM · Inference 推理与部署

## 知识点：KV Cache
### 定义
- 在解码阶段缓存历史 K/V，避免重复计算。
### 核心理解
- 用显存换速度，显著降低长上下文延迟。
### 关键细节
- 对首 token 后吞吐提升明显。
- 上下文越长，缓存占用越大。
### 例子 / 直觉
- 像保留中间算式，不必每次重算。
### 相关知识点
- `Attention` `推理优化`

## 知识点：量化技术
### 定义
- 用低位宽表示权重/激活以降低推理成本。
### 核心理解
- 在可接受精度损失下换取吞吐和成本优势。
### 关键细节
- 常见：INT8/INT4、PTQ/QAT。
- 要检查硬件支持与任务精度回退。
### 例子 / 直觉
- 类似图片压缩：体积更小，可能有轻微损失。
### 相关知识点
- `LoRA` `推理优化`

## 知识点：推理优化
### 定义
- 针对延迟、吞吐、成本做系统级优化。
### 核心理解
- 模型能力之外，服务系统决定最终体验。
### 关键细节
- 指标：TTFT、TPS、P95 延迟、成本/千 token。
- 手段：批处理、并行、算子融合、投机解码。
### 例子 / 直觉
- 同模型不同推理栈，性能可差一个量级。
### 相关知识点
- `KV Cache` `量化技术` `RAG`
