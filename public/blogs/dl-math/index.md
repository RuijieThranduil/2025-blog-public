# DeepLearning · Math 数学地基

很多人学深度学习时，会把数学当作一道门槛。  
但更准确地说，数学不是门槛，而是这门语言本身。

## 1. 线性代数为什么几乎无处不在
深度学习里最常见的运算，本质上都能写成矩阵和向量的变换：
- 输入是向量
- 权重是矩阵
- 特征空间里的投影、旋转、压缩都能用线代理解

所以如果你能自然地读：
`Wx + b`

你就已经在读模型的骨架了。

## 2. 导数和梯度决定模型怎么学
训练的目标不是“把结果算出来”，而是“知道该往哪改”。

导数回答的是局部变化率。  
梯度回答的是：在多维空间里，loss 上升最快的方向是什么。

于是我们才能反过来沿着负梯度方向去更新参数。

所以梯度不是某个附属概念，它就是 learning signal。

## 3. 概率论让模型从确定计算走向不确定世界
真实世界的数据带噪声、不完整、分布不稳定。  
概率论让我们能表达：
- 不确定性
- 条件关系
- 分布差异
- 期望风险

很多 loss、很多生成模型、很多推断方法，底层都离不开概率视角。

## 4. 特征值 / 特征向量为什么值得理解
它们看起来抽象，但在很多场景里其实很有直觉：
- 一个变换最重要的方向是什么
- 某个方向被拉伸了多少
- 数据的主变化轴在哪里

PCA、协方差分析、很多谱方法，背后都能看到特征值分解的影子。

## 5. 数学在这里的角色不是考试题，而是解释器
一个很重要的心态是：  
别把数学当作“先学完才能碰模型”的前置阻塞。

更好的方式通常是：
- 先碰模型
- 在卡住的地方补数学
- 用数学重新解释你刚看到的现象

这样数学会从抽象定义，变成有上下文的解释器。

## 6. 最值得优先补的几块
如果你的目标是更稳地读 deep learning 内容，优先级大概是：
- 线性代数
- 微积分与链式法则
- 概率统计
- 优化基础

这几块一旦通了，很多模型不再像“魔法”，而更像一套可推演的系统。

## One-line takeaway
深度学习不是绕开数学，而是把数学变成了可训练系统的工作语言。
