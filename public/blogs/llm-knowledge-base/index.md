## 模块 00 · MOC

## 知识点：LLM 生命周期总览

### 定义
- 从数据准备到预训练、微调、对齐、推理、应用、评估的一条完整工程链路。

### 核心理解
- 本质是“能力构建 + 约束优化 + 系统落地”的连续过程。
- 重要性在于帮助你定位每个技术点处于哪一层。

### 关键细节
- 主流程：数据 → 预训练 → SFT → 对齐 → 推理部署 → RAG/Agent → 评估。
- 每一层的输入输出都应可追踪。

### 例子 / 直觉
- 像软件流水线：先训练“通用引擎”，再做“场景调校”，最后做“线上运行”。

### 相关知识点
- `预训练范式` `SFT 监督微调` `RLHF` `推理优化` `评估方法`

## 知识点：Transformer 学习路径

### 定义
- 针对 Transformer 的分层学习顺序：Embedding → Attention → Block → 训练目标。

### 核心理解
- 先理解信息表示，再理解信息交互，最后理解参数更新。

### 关键细节
- 先掌握 token/位置编码，再理解 QKV 与多头注意力。
- 再看残差、归一化、前馈层如何稳定训练。

### 例子 / 直觉
- 先学“词怎么编码”，再学“词怎么互相看”，最后学“网络怎么堆叠”。

### 相关知识点
- `词向量与 Embedding` `Attention 机制` `Transformer 架构`

## 知识点：训练与对齐主线

### 定义
- 从“学会语言统计规律”到“学会按人类偏好输出”的两阶段主线。

### 核心理解
- 预训练解决“会不会”，对齐解决“像不像人要的答案”。

### 关键细节
- 预训练：下一词预测。
- 对齐：偏好数据 + 奖励建模或直接偏好优化。

### 例子 / 直觉
- 像先学语法，再学“在具体场景怎么说更合适”。

### 相关知识点
- `Scaling Law` `RLHF` `DPO` `Constitutional AI`

## 模块 01 · Foundations

## 知识点：语言模型基础

### 定义
- 语言模型学习序列概率分布，核心目标是预测下一个 token 概率。

### 核心理解
- 本质是用参数化函数逼近自然语言分布。
- 重要性：几乎所有 GPT 类模型都基于该目标。

### 关键细节
- 链式分解：\(P(x)=\prod_t P(x_t|x_{<t})\)
- 常用指标：Perplexity（困惑度）。

### 例子 / 直觉
- 给定“今天天气很”，模型给“好/冷/热”分配不同概率。

### 相关知识点
- `预训练范式` `Transformer 架构` `评估方法`

## 知识点：词向量与 Embedding

### 定义
- 将离散 token 映射到连续向量空间的表示方法。

### 核心理解
- 本质是把“符号”变成可计算的几何对象。
- 重要性：是神经网络处理文本的第一层入口。

### 关键细节
- 语义相近词通常向量距离更近。
- LLM 中常配合位置编码表示序列顺序。

### 例子 / 直觉
- 把“猫”“狗”放在高维空间里，二者距离通常比“猫”“火车”更近。

### 相关知识点
- `Attention 机制` `Transformer 架构` `RAG 基础`

## 知识点：Attention 机制

### 定义
- 通过 Query/Key 的相似度计算权重，对 Value 做加权聚合。

### 核心理解
- 本质是动态信息路由：每个 token 决定“该看谁”。
- 重要性：解决长距离依赖问题。

### 关键细节
- 缩放点积注意力：\(\text{softmax}(QK^T/\sqrt{d_k})V\)
- 多头机制可学习不同关系模式。

### 例子 / 直觉
- 句子中代词“它”会更关注其真实指代对象。

### 相关知识点
- `Transformer 架构` `KV Cache` `推理优化`

## 知识点：Transformer 架构

### 定义
- 以自注意力为核心的序列建模架构，替代传统 RNN/CNN 主干。

### 核心理解
- 通过并行注意力计算提升表达能力与训练效率。

### 关键细节
- 基础块：Multi-Head Attention + FFN + Residual + LayerNorm。
- Decoder-only 架构是当前主流 LLM 形态。

### 例子 / 直觉
- 像一组并行“阅读器”，分别关注语法、实体、上下文关系。

### 相关知识点
- `语言模型基础` `预训练范式` `推理优化`

## 模块 02 · Pretraining

## 知识点：数据处理与清洗

### 定义
- 对海量原始语料进行过滤、去重、规范化和质量控制。

### 核心理解
- 垃圾数据会直接污染模型能力上限。

### 关键细节
- 常见流程：采集 → 去重 → 质量过滤 → 安全过滤 → 混合采样。
- 关注重复率、语言分布、领域平衡。

### 例子 / 直觉
- 训练前先“洗米”，否则模型学到大量噪声模式。

### 相关知识点
- `预训练范式` `Scaling Law` `评估方法`

## 知识点：预训练范式

### 定义
- 在大规模无标注文本上，通过自监督目标学习通用语言能力。

### 核心理解
- 目标是得到可迁移的基础能力，而非某个单任务最优。

### 关键细节
- 常见目标：Causal LM、Masked LM。
- 数据规模、模型规模、训练步数相互耦合。

### 例子 / 直觉
- 像先学“通用语言常识”，再进入场景化训练。

### 相关知识点
- `语言模型基础` `Scaling Law` `SFT 监督微调`

## 知识点：Scaling Law

### 定义
- 描述模型性能与参数量、数据量、算力预算之间的幂律关系。

### 核心理解
- 不是“越大越好”，而是要在预算下找到最优配比。

### 关键细节
- 关注三元平衡：Model Size / Data Size / Compute。
- 常用于指导训练预算规划。

### 例子 / 直觉
- 像配方：只加面粉不加水（只增参数不增数据）并不会做出好面团。

### 相关知识点
- `预训练范式` `数据处理与清洗` `推理优化`

## 模块 03 · Finetuning

## 知识点：SFT 监督微调

### 定义
- 用高质量指令-答案样本继续训练，让模型更贴近任务输出格式。

### 核心理解
- 在“会说话”基础上，训练“按要求说话”。

### 关键细节
- 训练信号来自标准答案或专家示例。
- 对格式遵循、可用性提升明显。

### 例子 / 直觉
- 从自由写作转为“按模板作答”的训练。

### 相关知识点
- `指令微调` `RLHF` `DPO`

## 知识点：指令微调

### 定义
- 以 instruction-following 数据为核心的微调方式。

### 核心理解
- 目标是强化“任务理解 + 输出约束”能力。

### 关键细节
- 样本通常包含：指令、上下文、响应。
- 关注多任务覆盖和格式一致性。

### 例子 / 直觉
- 用户说“总结成三点”，模型能稳定遵守。

### 相关知识点
- `SFT 监督微调` `评估方法` `Constitutional AI`

## 知识点：LoRA 与 PEFT

### 定义
- Parameter-Efficient Finetuning：只更新少量参数完成任务适配。

### 核心理解
- 用更低显存/算力成本实现可接受效果。

### 关键细节
- LoRA 在权重更新中引入低秩分解矩阵。
- 适合多任务并行管理多个 adapter。

### 例子 / 直觉
- 不重做整台发动机，只加可拆卸“功能模块”。

### 相关知识点
- `SFT 监督微调` `推理优化` `量化技术`

## 模块 04 · Alignment

## 知识点：RLHF

### 定义
- 基于人类偏好数据训练奖励模型，再通过强化学习优化策略。

### 核心理解
- 让模型输出“更符合人类偏好”，不仅仅是“语言上合理”。

### 关键细节
- 典型流程：SFT → RM → PPO。
- 关注 reward hacking 与训练稳定性。

### 例子 / 直觉
- 给模型“评分老师”，让它逐步学会更高分答法。

### 相关知识点
- `DPO` `Constitutional AI` `评估方法`

## 知识点：DPO

### 定义
- Direct Preference Optimization：直接用偏好对优化策略，不显式训练奖励模型。

### 核心理解
- 以更简化训练流程替代传统 RLHF 复杂链路。

### 关键细节
- 使用“chosen vs rejected”样本对。
- 训练更稳定，工程复杂度更低。

### 例子 / 直觉
- 直接比较“哪个答案更好”，而不是先学打分器再学策略。

### 相关知识点
- `RLHF` `SFT 监督微调` `Constitutional AI`

## 知识点：Constitutional AI

### 定义
- 通过一套明确“宪法原则”指导模型自我批评与自我修正。

### 核心理解
- 把安全与价值约束显式化、规则化。

### 关键细节
- 原则可覆盖安全、偏见、合规等维度。
- 可减少对高成本人工标注的依赖。

### 例子 / 直觉
- 给模型一本“行为准则手册”，回答前后都要对照检查。

### 相关知识点
- `RLHF` `DPO` `评估方法`

## 模块 05 · Inference

## 知识点：KV Cache

### 定义
- 在自回归解码中缓存历史 token 的 Key/Value 以避免重复计算。

### 核心理解
- 用空间换时间，显著降低长上下文生成延迟。

### 关键细节
- 对吞吐和首 token 之后的速度提升明显。
- 需要关注显存占用随上下文增长。

### 例子 / 直觉
- 像考试时把已算过的中间结果放草稿纸，不必每题重算。

### 相关知识点
- `Attention 机制` `推理优化` `量化技术`

## 知识点：量化技术

### 定义
- 使用低位宽（如 INT8/INT4）表示权重或激活，降低推理成本。

### 核心理解
- 在精度可接受前提下换取更低显存与更高吞吐。

### 关键细节
- 方案：PTQ、QAT、AWQ、GPTQ 等。
- 需评估任务精度退化和硬件支持。

### 例子 / 直觉
- 把高清图压缩到较小体积，画质略降但传输更快。

### 相关知识点
- `KV Cache` `推理优化` `LoRA 与 PEFT`

## 知识点：推理优化

### 定义
- 针对在线服务的延迟、吞吐、成本进行系统级优化。

### 核心理解
- 模型能力只是起点，服务质量取决于系统工程。

### 关键细节
- 常见手段：批处理、并行、speculative decoding、算子融合。
- 指标：TTFT、TPS、P95 延迟、成本/千 token。

### 例子 / 直觉
- 同一模型，不同推理栈可出现数量级性能差距。

### 相关知识点
- `KV Cache` `量化技术` `Agent 框架`

## 模块 06 · RAG & Agents

## 知识点：RAG 基础

### 定义
- 检索增强生成：先检索外部知识，再结合上下文生成答案。

### 核心理解
- 通过外部知识库缓解知识截止与幻觉问题。

### 关键细节
- 流程：切分 → 向量化 → 检索 → 重排 → 生成 → 引用。
- 关键指标：检索召回率、答案忠实度、引用准确率。

### 例子 / 直觉
- 像开卷考试：先翻资料，再组织答案。

### 相关知识点
- `Tool Use` `评估方法` `Benchmark 汇总`

## 知识点：Tool Use

### 定义
- 让模型调用外部工具（搜索、计算、数据库、API）完成任务。

### 核心理解
- 把模型从“纯文本生成器”升级为“可执行任务代理”。

### 关键细节
- 关键环节：工具选择、参数构造、结果校验、异常回退。
- 需防止工具调用链无限循环。

### 例子 / 直觉
- 模型先调日历 API 查时间，再生成行程建议。

### 相关知识点
- `Agent 框架` `RAG 基础` `Constitutional AI`

## 知识点：Agent 框架

### 定义
- 将规划、记忆、工具调用、反思等机制组织成可迭代执行系统。

### 核心理解
- 重点是“决策闭环”，不是一次性回答。

### 关键细节
- 常见模式：ReAct、Plan-and-Execute、多 Agent 协同。
- 要有状态管理与终止条件。

### 例子 / 直觉
- 像项目助理：先拆任务，再查资料，再执行，再复盘。

### 相关知识点
- `Tool Use` `RAG 基础` `评估方法`

## 模块 07 · Evaluation

## 知识点：评估方法

### 定义
- 对模型能力、安全性、稳定性和成本进行系统化测量的方法集合。

### 核心理解
- 不评估就无法迭代；评估标准决定优化方向。

### 关键细节
- 维度：能力（准确率）、鲁棒性、安全性、延迟、成本。
- 方法：自动评测 + 人工评测 + 线上 A/B。

### 例子 / 直觉
- 离线考试高分，不代表线上服务一定好用。

### 相关知识点
- `Benchmark 汇总` `RAG 基础` `Constitutional AI`

## 知识点：Benchmark 汇总

### 定义
- 用标准数据集与任务基准对模型进行可比评测。

### 核心理解
- Benchmark 提供横向比较，但不等于真实业务表现。

### 关键细节
- 关注：数据污染、任务覆盖、是否过拟合榜单。
- 应结合场景自定义评测集。

### 例子 / 直觉
- 体育联赛排名能反映实力，但不代表每种场地都必赢。

### 相关知识点
- `评估方法` `Scaling Law` `RAG 基础`
