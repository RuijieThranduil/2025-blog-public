# DeepLearning · Optimization 优化与收敛

训练深度学习模型时，我们嘴上说的是“在学”，本质上做的却是 optimization。

## 1. 训练为什么会像在山谷里找路
loss surface 可以粗暴理解成一张地形图。  
参数在高维空间里移动，目标是往更低的区域走。

困难在于这张图通常：
- 非凸
- 有 saddle point
- 局部区域尺度差异很大
- 噪声很多

所以优化器的工作，不只是“往下走”，而是“在复杂地形里高效又别翻车地往下走”。

## 2. SGD 为什么到现在还这么重要
最基本的更新式子就是：

`theta = theta - lr * grad`

它简单到几乎朴素，但仍然是很多优化讨论的 baseline。

SGD 的好处：
- 简单
- 泛化 often surprisingly good
- 对很多训练技巧兼容性强

它的缺点也很直接：
- 收敛路径抖
- 不同方向尺度不一致时比较难走

## 3. Momentum：给优化加惯性
如果把 SGD 看成每一步都“重新决定往哪走”，Momentum 更像在保留一部分历史速度。

直觉上它有两个作用：
- 沿着正确方向走得更稳
- 减少来回震荡

所以在狭长 valley 里，Momentum 往往比原始 SGD 更顺。

## 4. Adam 为什么工程里这么常见
Adam 可以理解成：
- 有 Momentum 的一阶动量
- 再加上对梯度尺度的自适应调整

所以它常常带来：
- 更快的前期收敛
- 对超参数更宽容一点
- 在复杂任务上更容易先跑起来

但这不代表 Adam 永远最好。  
有些任务最终泛化上，SGD + Momentum 仍然很能打。

## 5. 优化难，不一定是优化器本身的问题
很多时候我们看到 loss 卡住，会下意识怪优化器。  
但真实原因可能是：
- 学习率不合适
- 初始化差
- normalization 没配好
- loss 设计有问题
- 数据噪声太大

优化器只是系统的一部分，不是万能补丁。

## 6. 如何读一个训练曲线
几个常见信号：
- loss 平稳下降：通常是健康状态
- 高频震荡：lr 可能偏大
- 很慢但在降：lr 可能偏小
- 突然 NaN：数值稳定性、混合精度或梯度爆炸要查

工程里会读曲线，比会背优化器公式更重要。

## 7. 一个实用心法
别把 optimizer 当“黑盒默认设置”。  
更好的思路是问：
- 当前任务的梯度是不是 noisy
- 模型是不是很深
- 收敛速度和最终泛化哪个更重要

不同任务，optimizer preference 会不一样。

## One-line takeaway
优化器不是训练里的配角，它决定了模型是跌跌撞撞地下山，还是能稳定找到一条真正可收敛的路。
