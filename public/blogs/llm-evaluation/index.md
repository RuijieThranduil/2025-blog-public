# LLM · Evaluation 评估

## 知识点：评估方法
### 定义
- 系统评估模型能力、安全、鲁棒性、成本的方法集合。
### 核心理解
- 不评估无法迭代；评估指标决定优化方向。
### 关键细节
- 维度：准确性、安全性、鲁棒性、延迟、成本。
- 方法：离线评测 + 人工评测 + 线上 A/B。
### 例子 / 直觉
- 离线高分不代表线上体验一定好。
### 相关知识点
- `Benchmark` `RAG`

## 知识点：Benchmark 汇总
### 定义
- 用标准任务与数据集做横向比较。
### 核心理解
- Benchmark 是参考，不是业务真实效果的全部。
### 关键细节
- 关注数据污染、任务覆盖和榜单过拟合。
- 建议叠加场景自建评测集。
### 例子 / 直觉
- 联赛排名有价值，但不等于任何场地都必赢。
### 相关知识点
- `评估方法` `Scaling Law`
