# DeepLearning · NLP 表示与建模

NLP 最大的难点之一，是语言既离散又含糊。  
深度学习能进入 NLP，一个关键前提就是：先把词和句子变成可计算的表示。

## 1. Embedding 改变了什么
在 embedding 之前，词更像离散 ID。  
有了 embedding 之后，词开始进入连续向量空间。

这件事意义很大，因为它让模型能表达：
- 语义相似
- 上下文关系
- 某种几何上的结构邻近

简单说，embedding 是 NLP 深度学习化的入口。

## 2. 序列模型为什么适合语言
语言不是无序集合，而是有顺序、有依赖的 sequence。

所以早期 NLP 深度学习会大量使用：
- RNN
- LSTM
- GRU

它们很适合表达：
- 前文影响后文
- 长短期记忆
- 上下文累积

## 3. 语言任务其实不止一种
常见任务可以很粗地分成几类：
- classification：情感分析、主题分类
- sequence labeling：NER、词性标注
- seq2seq：翻译、摘要
- language modeling：预测下一个 token

任务不同，建模方式和损失设计也不同。

## 4. Attention 为什么对 NLP 特别关键
语言有一个天然难点：  
远距离依赖很多时候真的很远。

比如一句很长的话里，主语和真正控制语义的词可能隔了很多 token。

Attention 的价值就在于：
- 不需要只靠“最后一个 hidden state”记住全部上下文
- 可以直接对相关位置做动态聚合

这让很多长依赖问题不再那么拧巴。

## 5. 从传统 NLP 到大模型，中间其实有连续主线
今天我们看 Transformer 和 LLM，容易把它们当成完全新世界。  
但它们和 earlier NLP deep learning 之间其实有很连续的脉络：
- 先做词表示
- 再做上下文建模
- 再提升长依赖处理能力
- 最后把建模规模做大

所以旧方法不是“过时垃圾”，而是 current paradigm 的台阶。

## 6. 一个适合入门的理解框架
读 NLP 模型时，可以先问三件事：
- 文本被表示成什么
- 上下文是怎么被整合的
- 输出目标是什么

这三个问题一旦清楚，大多数结构就不会显得玄。

## One-line takeaway
NLP 和深度学习结合的本质，是把语言从离散符号问题，转成了表示学习和上下文建模问题。
