# DeepLearning · Detection 目标检测

目标检测是 CV 里非常“工程味”的任务，因为它要求模型同时回答两个问题：
- 这是什么
- 它在哪里

## 1. Detection 比分类多出来了什么
分类只需要给整张图一个标签。  
检测则要对图中多个对象输出：
- 类别
- 位置框
- 置信度

所以它不是单纯更复杂，而是任务目标发生了变化。

## 2. Proposal 思路解决了搜索空间太大问题
如果直接在整张图的所有位置、所有尺度上穷举，代价太大。

于是早期 two-stage 检测器会先提 region proposal，  
再在候选区域上做分类和框回归。

这个思路的价值在于：  
先缩小搜索空间，再做更精细判断。

## 3. Anchor 是一种任务工程化妥协
Anchor 的直觉是：  
先给不同位置放一些预设大小和比例的参考框，再让模型去修正。

这样做好处是训练目标更稳定。  
代价是：
- 设计更复杂
- 超参数更多
- 正负样本分配比较讲究

后来很多 anchor-free 方法，其实就在试图把这套先验做得更轻。

## 4. IoU 为什么是检测里的核心语言
检测不是只看“有没有框”，还要看框得准不准。

IoU 用来描述：

`预测框 和 真实框 的重叠程度`

它会参与：
- 正负样本判断
- NMS
- 评估指标

所以如果你在读 detection paper，IoU 基本是必须会说的语言。

## 5. One-stage 与 Two-stage 的直觉区别
Two-stage：
- 更像“先找候选，再精修”
- 往往精度强

One-stage：
- 更像“直接在密集位置上做预测”
- 往往速度更好，部署更直接

YOLO 系列之所以火，很大一部分原因就在于它把检测做成了更统一、更实时的 pipeline。

## 6. Detection 真正难的不是写个框，而是处理现实世界
真实数据里会有很多 hard case：
- 小目标
- 遮挡
- 密集目标
- 类别不平衡
- 背景复杂

这也是为什么 detection 的改进经常围绕：
- multi-scale
- label assignment
- loss design
- data augmentation

## One-line takeaway
目标检测的核心不是“多画几个框”，而是把分类、定位和密集预测这三件事，压进一个可以稳定训练和高效部署的系统里。
