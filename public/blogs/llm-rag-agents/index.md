# LLM · RAG & Agents

## 知识点：RAG 基础
### 定义
- 检索增强生成：先检索外部知识，再基于检索结果生成。
### 核心理解
- 缓解知识截止和幻觉，提升可追溯性。
### 关键细节
- 流程：切分 → 向量化 → 检索 → 重排 → 生成 → 引用。
- 关键指标：召回率、忠实度、引用准确率。
### 例子 / 直觉
- 开卷考试：先查资料再答题。
### 相关知识点
- `评估方法` `Tool Use`

## 知识点：Tool Use
### 定义
- 让模型调用外部工具/API完成任务。
### 核心理解
- 把模型从“文本生成器”变成“任务执行器”。
### 关键细节
- 工具选择、参数生成、结果验证、异常回退。
- 防止无终止循环调用。
### 例子 / 直觉
- 调天气 API 后再回答穿衣建议。
### 相关知识点
- `Agent 框架` `Constitutional AI`

## 知识点：Agent 框架
### 定义
- 组织规划、执行、反思、记忆的闭环系统。
### 核心理解
- 重点是多步决策，不是一次性回答。
### 关键细节
- 常见模式：ReAct、Plan-and-Execute。
- 需状态管理与终止条件。
### 例子 / 直觉
- 像项目助理：拆任务、查资料、执行、复盘。
